## 1. なぜ定性分析が必要か

### inga-quantsの限界

inga-quantsは優秀な定量エンジンだが、以下を捕捉できない:

- **レジーム変化の初動** — 数値に表れる前のマクロ転換（例: FRBの姿勢変化）
- **機関投資家の意図** — ポジション構築中の思惑は価格に遅れて反映される
- **イベントの非線形リスク** — SQ週、FOMC前後、地政学リスクは統計的に扱いにくい
- **ナラティブの力** — 「AI相場」「利下げ期待」等の物語が資金フローを生む
- **事実の連鎖** — 単日のニュースではなく、数日〜数週間にわたる物語の「流れ」

**結論: 定量スコアの上に「今日これを信じていいのか」を判断する層が必要。**

## 3. ナラティブ・スレッド（物語の糸）

### 3.1 問題: 点ではなく線を読む

ニュースを日単位で処理すると文脈が消える。

```
月曜: 「パウエル議長、利下げ示唆」        → 事実として処理、忘れる
水曜: 「インフレ指標が予想上振れ」        → 事実として処理、忘れる
金曜: 「FRB高官が利下げ慎重発言」        → 「慎重だな」で終わる

↑ 月→水→金の流れを見れば
 「月曜の利下げ期待が完全に剥落。ポジション巻き戻しが来る」
  これがヘッジファンドが見ている世界。
```

### 3.2 ナラティブ・スレッドDB

```sql
CREATE TABLE narrative_threads (
    thread_id TEXT PRIMARY KEY,          -- "fed_rate_2026q1"
    theme TEXT,                          -- "FRB利下げ観測"
    started_date TEXT,                   -- 最初に検知した日
    last_updated TEXT,                   -- 最後に更新した日
    status TEXT,                         -- active / resolved / reversed
    
    -- 時系列の事実チェーン（JSON配列）
    fact_chain TEXT,                     -- [
                                         --   {"date":"03-24","fact":"パウエル利下げ示唆","direction":"+"},
                                         --   {"date":"03-26","fact":"CPI上振れ","direction":"-"},
                                         --   {"date":"03-28","fact":"高官慎重発言","direction":"-"}
                                         -- ]
    
    current_direction TEXT,              -- strengthening / weakening / reversed
    market_impact_so_far TEXT,           -- これまでの市場への影響
    affected_sectors TEXT,               -- 影響を受けるセクター（JSON配列）
    confidence REAL                      -- このスレッドの重要度 0-100
);
```

### 3.3 スレッドが特徴量評価に与える影響

```
# スレッド情報なし（旧設計）
M2: 金融政策スタンス = 引締め寄り
    根拠: 高官の慎重発言

# スレッド情報あり（新設計）
M2: 金融政策スタンス = 引締め寄り
    根拠: fed_rate_2026q1スレッドが3/24の利下げ示唆から
    3日で反転。市場はまだ利下げ織込み中なので
    剥落リスク大。ポジション巻き戻しに警戒。
    
→ 同じ「引締め寄り」でも深さが全然違う
```

### 3.4 スレッドのライフサイクル

```
[誕生] 新しいテーマの事実を検知 → 新規スレッド作成
  ↓
[成長] 後続の事実が同じ方向を支持 → direction: strengthening
  ↓
[転換] 逆方向の事実が出現 → direction: weakening / reversed
  ↓  ← ここがトレード機会（ポジション巻き戻し）
[解決] テーマが市場に織り込まれた / 結論が出た → status: resolved
  ↓
[消滅] 3日以上更新なし → status: resolved（自然消滅）
```

## 4. ニュース処理の4段パイプライン

### 4.1 なぜニュースをそのまま読ませてはいけないか

- ヘッドラインの感情に引っ張られる（「暴落」「急騰」に過剰反応）
- 同じ事実でもソースの書き方で判断がブレる
- 「多く報道されてる＝重要」と誤認する
- 逆張りすべき局面で順張り分析になる（全員弱気の時こそ買い場）

### 4.2 4段パイプライン

```
[06:30] 生ニュース取得
    │
    ▼
[Call #1] 事実抽出（感情排除）
    │ 入力: 生ヘッドライン群
    │ 出力: ファクトシート（感情・予測・形容詞を排除した事実のみ）
    │
    │ prompt: 「以下のヘッドライン群から事実だけを抽出せよ。
    │          感想・予測・形容詞は除外。」
    │ 出力例: {"facts": [
    │   {"id": "F1", "what": "FRBが金利据え置き", "category": "monetary_policy"},
    │   {"id": "F2", "what": "トヨタ Q3 営業利益+15%", "category": "earnings"}
    │ ]}
    ▼
[Call #2] ナラティブ・スレッド更新
    │ 入力: 今日の事実 + 既存アクティブスレッド群
    │ 出力: 更新されたスレッド群
    │
    │ prompt: 「各事実が既存スレッドの続報か判定。
    │          紐付く場合→fact_chainに追加、方向判定。
    │          新規テーマ→スレッド作成。
    │          3日更新なし→resolved検討。
    │          反転したスレッドには特に注意。」
    │ 出力例: {"updated_threads": [...], "new_threads": [...], "resolved": [...]}
    ▼
[Call #3] 28特徴量評価
    │ 入力: ファクトシート + 定量データ + アクティブスレッド群
    │ 条件: 各特徴量に「根拠となる事実ID or スレッドID」を必須
    │
    │ prompt: 「事実と数値のみに基づき28特徴量を評価せよ。
    │          ヘッドラインの感情的表現には影響されないこと。
    │          各特徴量に根拠(事実番号 or スレッドID)を必ず付与。」
    ▼
[Call #4] コンセンサス逆張りチェック
    │ 入力: 28特徴量の評価結果
    │
    │ prompt: 「この分析は市場コンセンサスと同じ方向か？
    │          全員が同じ方向を向いている場合、逆シナリオは何か？
    │          逆シナリオの確率は？
    │          全員弱気→底打ちの可能性。全員強気→天井の可能性。」
    │
    │ 出力: contrarian_flag (bool), contrarian_scenario (text),
    │       contrarian_probability (0-100)
    ▼
[最終評価JSON] → DB保存 → 朝ブリーフィング
```

## 5. 決算書の扱い

### 5.1 方針: 数字はプロに任せ、律は文脈を読む

決算書の数字解析はサブスクサービスが安くて正確。
Claude Visionで100銘柄のPDFを読むと$10/日。サービスに任せるべき。

```
[サブスク/既存サービス] → 数字の要約を構造化データで取得
  - 株探: 決算速報 + サプライズ度
  - バフェット・コード: 財務比較
  - マネックス銘柄スカウター: セクター横断分析

        ↓ 構造化された数字だけ受け取る

[律の仕事] → 「この決算がナラティブ・スレッドにどう影響するか」
  - トヨタ好決算 → 「円安恩恵ナラティブが強化される」
  - 半導体ガイダンス下方修正 → 「AI投資スレッドに亀裂」
  - 小売り軒並み弱い → 「内需セクター回避のシグナル」
```

### 5.2 決算シーズンの律の処理

```python
# 決算速報データ（サブスクやスクレイピングで取得済み）
earnings_data = {
    "ticker": "7203",
    "name": "トヨタ",
    "quarter": "Q3 FY2026",
    "revenue_surprise": "+3.2%",  # コンセンサス対比
    "profit_surprise": "+8.1%",
    "guidance_change": "上方修正",
    "key_factor": "円安効果 + 北米好調"
}

# 律に聞くのは「数字」ではなく「意味」
prompt = f"""
## 決算結果（構造化済み）
{earnings_data}

## 現在アクティブなナラティブ・スレッド
{active_threads}

## 指示
1. この決算はどのナラティブ・スレッドに影響するか？
2. スレッドは強化されるか？弱体化するか？
3. セクター全体への波及はあるか？
4. 明日以降の売買判断に影響する洞察は？

※ 数字の分析は不要。文脈と影響のみ回答せよ。
"""
```

### 5.3 コスト比較

```
❌ 律が決算書PDF解析: $0.05-0.10/銘柄 × 100銘柄/日 = $5-10/日
✅ サブスク + 律は文脈のみ: サブスク月額 + $0.01/銘柄 = 微小
```

### 6.1 日次サイクル

```
  ▼

  ▼

  │ 「今日はFOMC前で様子見ムードかも…」
  │ 「FRB利下げスレッドが反転してきてるから注意です」
  ▼

  │ 当日の日経平均騰落率, セクター別成績
  ▼

```

### 6.2 進化メカニズム（Few-Shot学習）

**核心: 律の過去の「当たった分析」をsystem promptに注入する。**

```python
def build_qualitative_prompt():
    # 直近のデータ
    news = fetch_news()
    indicators = fetch_indicators()
    active_threads = db.get_active_threads()
    
    # 過去の「当たった分析」上位5件を取得
    best_analyses = db.query("""
        SELECT date, features_json, ritsu_comment, accuracy_score
        FROM qualitative_evaluations
        WHERE accuracy_score >= 75
        ORDER BY date DESC LIMIT 5
    """)
    
    # 過去の「大きく外した分析」上位3件（反面教師）
    worst_analyses = db.query("""
        SELECT date, features_json, ritsu_comment, reflection
        FROM qualitative_evaluations
        WHERE accuracy_score <= 30
        ORDER BY date DESC LIMIT 3
    """)
    
    # 類似局面の分析（スレッドテーマが近い過去の日）
    similar_analyses = db.query("""
        SELECT date, features_json, ritsu_comment, accuracy_score
        FROM qualitative_evaluations
        WHERE features_json LIKE '%{dominant_theme}%'
        AND accuracy_score IS NOT NULL
        ORDER BY accuracy_score DESC LIMIT 3
    """)
    
    return f"""
## 過去に精度が高かった分析例
{format_examples(best_analyses)}

## 過去に外した分析と反省
{format_examples(worst_analyses)}

## 類似局面の過去分析
{format_examples(similar_analyses)}

## アクティブなナラティブ・スレッド
{format_threads(active_threads)}

## 今日のデータ
{news}
{indicators}

上記を踏まえ、28特徴量を評価し、総合判断を出してください。
"""
```

### 6.4 進化のフェーズ

```
Phase 1 (1-2週目): 蓄積期
  - 毎日28特徴量+スレッドを記録するだけ
  - few-shotなし（データがまだない）
  - 律「まだ勉強中です…」

Phase 2 (3-4週目): 学習開始
  - 10日分以上のデータ蓄積
  - 成功/失敗パターンのfew-shot注入開始

  - 律「最近ちょっとわかってきたかも…」

Phase 3 (2ヶ月目〜): パターン認識
  - 40日分以上のデータ
  - 曜日効果、SQ週パターン、FOMC前後パターン等の傾向が見える
  - 類似局面検索が機能し始める
  - 律「SQ週は毎回こうなるんだよね…」

Phase 4 (3ヶ月目〜): inga-quants統合
  - 定量スコアと定性スコアの統合
  - 「数字は買いだけど、この地合いでは見送り」が可能に
  - 律の判断がバックテスト可能になる
```

## 7. inga-quantsとの統合

### 7.1 統合スコア

```
最終スコア = α × 定量スコア + (1-α) × 定性スコア

αの決め方:
- デフォルト: 0.7 (定量7 : 定性3)
- イベント日: 0.5 (定性の重みを上げる)
- 決算シーズン: 0.6
- αは定性分析の精度が上がるにつれ自動調整
```

### 7.2 シグナルフィルタとしての定性分析

```python
# inga-quants出力
quant_signal = {"ticker": "7203", "score": 0.82, "confidence": "high"}

# 律の定性フィルタ
qualitative = {
    "overall_stance": "defensive",
    "event_risk": "high",               # FOMC前日
    "institutional_intent": -1,          # 海外勢売り
    "narrative_risk": "fed_rate反転中",  # スレッドから
}

# 統合判断
if qualitative["overall_stance"] == "defensive":
    ritsu_comment = "7203すごく良いんだけど…FOMC前だし、今日は様子見にしません？"
```

## 設計原則

1. **型が本体** — Claudeの分析力ではなく、28特徴量の構造が価値
2. **毎日同じ型** — 一貫したフォーマットで蓄積するから比較・学習できる
3. **点ではなく線** — ナラティブ・スレッドで事実の連鎖を追跡する
4. **事実と感情を分離** — ニュースの感情に引っ張られない4段パイプライン
5. **自己採点** — 答え合わせを自動化して律自身が精度を意識する
6. **few-shotが進化** — 蓄積された成功/失敗パターンがpromptを進化させる
7. **数字はプロに任せる** — 決算書解析はサブスク、律は文脈の解釈に集中
8. **人間が最終判断** — 律は「分析と意見」を出す。「買え/売れ」は絶対言わない
