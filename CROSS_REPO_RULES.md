# 因果quants ファミリー — リポ間運用ルール

> 全リポジトリ共通。セッション開始時に必ず確認すること。

## ファイル構成（全リポ共通）

```
リポジトリ/
├── CLAUDE.md           # 基本メモ（責務・公開API・依存先・設計原則）
├── PROJECT_STATUS.md   # 進行メモ（現在の状態・残タスク・直近変更）
├── CROSS_REPO_RULES.md # このファイル（運用ルール）
├── health.json         # 監査マニフェスト（ヘルスチェック用）
└── mailbox/            # 私書箱（他リポとの通信）
    ├── from-{送信元}_YYYYMMDD_{件名}.md
    └── done/           # 対応済み（アーカイブ）
```

## 編集権限

| ファイル | 自リポセッション | 他リポセッション |
|----------|:---:|:---:|
| CLAUDE.md | ✅ 編集可 | ❌ 読み取りのみ |
| PROJECT_STATUS.md | ✅ 編集可 | ❌ 読み取りのみ |
| health.json | ✅ 編集可 | ❌ 読み取りのみ |
| mailbox/ | ✅ 読み取り+対応済み移動 | ✅ **新規投函のみ** |

## セッション開始フロー

1. 自リポの`CLAUDE.md` + `PROJECT_STATUS.md`を読む
2. `mailbox/`を確認 → 未対応の依頼があれば優先度判断
3. 必要に応じて依存先リポの`CLAUDE.md`（公開APIセクションのみ）を参照
4. 司令官の指示に従い作業開始

## 私書箱（mailbox/）ルール

### 投函（他リポへ書く）
- ファイル名: `from-{自リポ短縮名}_YYYYMMDD_{件名}.md`
  - 短縮名: hp, quants, fact, kogane, ritsu
- 内容テンプレート:
  ```markdown
  # {件名}
  - 送信元: inga-{xxx}
  - 日付: YYYY-MM-DD
  - 種別: 要望書 / 報告書 / 仕様変更通知 / バグ報告
  - 緊急度: 高 / 中 / 低

  ## 概要
  （何が起きたか / 何をしてほしいか）

  ## 詳細
  （技術的な変更内容、影響範囲）

  ## 対応してほしいこと
  （具体的なアクション）
  ```

### 受信（自リポで読む）
- 対応完了 → `mailbox/done/`に移動
- 対応不要 → 削除 or `done/`に移動（理由をコミットメッセージに記載）
- 対応中 → ファイル先頭に`<!-- STATUS: 対応中 -->`を追加

### git操作
- 投函時: 相手リポにgit clone → mailbox/に追加 → commit+push
- コミットメッセージ: `mailbox: {送信元}から{件名}`

## 重要な設計原則（全リポ共通）

1. **推測禁止**: リポジトリやデータソースを確認してから返信
2. **スコア独立性**: factデータとquantsスコアは完全分離して表示
3. **専門用語禁止（ユーザー向け）**: LightGBM/z-score/σ等は表示しない
4. **non-fatal設計**: 補助データの取得失敗でパイプライン全体を止めない
5. **health.json更新**: 新機能追加時は監査マニフェストも更新すること
